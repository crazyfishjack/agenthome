# RAG Knowledge Base

RAG知识库管理技能，提供完整的文档检索、索引管理、分块处理和RAG检索召回重排功能。

## 功能说明

1. **文档索引管理**：自动生成和更新分类目录索引
2. **知识库查询**：返回所有分类和指定分类的目录索引
3. **向量数据库管理**：自动将文档分块并存储到向量数据库
4. **RAG检索**：支持完整的召回+重排检索流程

## 使用约束

> **重要约束**：

禁止臆想，当用户模糊的说文档名字时，需要查询，查询后严格按照查询到的文件名匹配

传入文件名或分类名时，必须严格按照已有的分类或文件名，不要加工出现多余的空格等情况，只改skill即可，很简短的约束即可。在读取到文件名或分类名时也要严格使用已有文件名。使用查询数据库信息和查询目录索引时，需要严格使用工具返回的文件名不要加工。

> **示例**：工具返回信息：
```json
{
  "documents": [
    {
      "doc_name": "产品A用户手册.txt",
      "doc_type": "txt",
      "update_time": "2026-03-09 17:30:18",
      "summary": "在此输入摘要"
    },
}
```

```
产品A用户手册.txt
```
不能理解为
```
产品 A 用户手册.txt
```
在使用时，需要严格使用“产品A用户手册.txt”这个原名称，禁止使用“产品 A 用户手册.txt”（禁止有多余的加工例如添加多余的空格，严格禁止）
禁止说：找到了"产品 A 用户手册.txt"文档。（和原名称相比加入了多余的空格）
应按说：找到了“产品A用户手册.txt”文档。

## 使用方法

### 1. 生成分类索引

当用户需要生成或更新某个分类的目录索引时：

```
execute_script(skill_id="rag_knowledge_base", script_name="generate_index", args=["--category", "金融分析"])
```

**参数说明：**
- `--category`：分类名称（必需）


**功能特性：**
- **支持的文档格式**：PDF（.pdf）、Word（.doc/.docx）、Excel（.xls/.xlsx）、TXT（.txt）、Markdown（.md）
- **自动提取文档信息**：文档名称、路径、类型、更新时间
- **摘要字段预留**：summary字段默认填充"在此输入摘要"，需要手动编辑填写
- **编码自适应**：TXT文件支持多种编码格式（utf-8、gbk、gb2312、utf-16），Markdown文件使用utf-8编码
- **错误处理**：当分类目录不存在时，返回友好的错误信息

**使用场景：**
1. 新增文档后更新索引
2. 定期维护知识库索引
3. 批量导入文档后建立索引
4. 手动编辑文档摘要


### 2. 查询知识库信息

#### 查询所有分类

**功能说明：**
返回知识库中所有分类的基本信息，包括分类名称、文档数量和最后更新时间。

**调用方式：**
```
execute_script(skill_id="rag_knowledge_base", script_name="query_knowledge_base")
```

**功能特性：**
- **自动扫描**：自动扫描知识库目录下的所有分类
- **索引读取**：读取每个分类的index.json文件获取详细信息
- **容错处理**：如果某个分类的index.json文件不存在或格式错误，仍然会列出该分类（文档数量为0）
- **错误提示**：如果知识库目录不存在，返回明确的错误信息

**使用场景：**
1. 浏览知识库整体结构
2. 查看各分类的文档数量
3. 了解知识库的更新状态
4. 为用户提供分类导航


#### 查询指定分类的目录索引

**功能说明：**
返回指定分类的详细文档列表，包括文档名称、类型、更新时间和摘要信息。

**调用方式：**
```
execute_script(skill_id="rag_knowledge_base", script_name="query_knowledge_base", args=["--category", "金融分析"])
```

**参数说明：**
- `--category`：分类名称（必需）


**功能特性：**
- **详细信息**：返回文档的名称、类型、更新时间和摘要
- **分类信息**：同时返回分类名称和分类的更新时间
- **路径隐藏**：不返回文档的完整路径，保护内部结构
- **摘要展示**：展示文档摘要（需手动编辑填写）

**使用场景：**
1. 查看某个分类下的所有文档
2. 通过摘要快速了解文档内容
3. 为用户提供文档选择界面
4. 检索前的文档筛选

### 3. 更新向量数据库

#### 功能说明

更新向量数据库脚本用于将文档分块并存储到Chroma向量数据库中，支持按分类批量更新或按单个文档更新。该脚本包含完整的文档处理流程：文档提取、文本清洗、智能分块、向量生成和数据库存储。

**核心功能：**
- **文档清洗**：只去除多余空行和空白字符,保留所有有用信息(页码、页眉页脚、水印、特殊符号等)
- **智能分块**：支持自定义分块大小和重叠字符数，智能识别句子边界
- **向量生成**：使用阿里云text-embedding-v4模型生成高质量向量
- **数据库管理**：自动删除旧分块，添加新分块，保持数据一致性
- **路径自动处理**：自动拼接完整路径，用户只需提供分类名称或文档名称

**向量数据库元数据结构：**
```json
{
  "chunk_id": "分类_文档名_003",
  "category": "分类名称",
  "doc_name": "文档名称.txt",
  "doc_update_time": "2026-03-08 10:00:00",
  "chunk_content": "分块内容",
  "chunk_token_size": 289,
  "chunk_index": 3,
  "doc_path": "rag/rag_document_lib/分类/文档名称.txt",
  "create_time": "2026-03-09 15:25:00"
}
```

#### 更新整个分类

**调用方式：**
```
execute_script(skill_id="rag_knowledge_base", script_name="update_vector_db", args=["--category", "金融分析", "--chunk_size", "300", "--chunk_overlap", "40"])
```

**参数说明：**
- `--category`：分类名称（必需，只需要分类名称，不需要完整路径）
- `--chunk_size`：分块字符大小（可选，默认300）
- `--chunk_overlap`：分块重叠字符数（可选，默认40）

**处理流程：**
1. 读取分类的index.json文件，获取文档列表
2. 删除该分类下所有旧的分块
3. 对每个文档执行：
   - 提取文本内容
   - 清洗文本（去除无用内容）
   - 智能分块（按chunk_size和chunk_overlap）
   - 生成embedding向量（使用text-embedding-v4模型）
   - 构建元数据
4. 批量添加新分块到Chroma向量数据库

**使用场景：**
- 新增分类后初始化向量数据库
- 批量更新分类下所有文档
- 修改分块参数后重新生成向量
- 定期维护和优化向量数据库

#### 更新单个文档

**调用方式：**
```

```

**参数说明：**
- `--document`：文档名称（必需，只需要文档名称，不需要完整路径）

- `--chunk_size`：分块字符大小（可选，默认300）

**使用场景：**
- 单个文档内容更新后重新生成向量

- 快速更新少量文档
- 调试和验证文档处理流程

**处理流程：**
1. 在所有分类目录中查找指定的文档
2. 删除该文档的所有旧分块

### 4. RAG检索召回重排

#### 重要说明：检索前的准备工作

在调用RAG检索脚本之前，**必须先执行以下步骤**以确保检索的准确性和全面性：

1. **调用查询知识库信息脚本**：先调用`query_knowledge_base`脚本获取知识库的最新分类和文档信息
2. **分析相关文档**：根据查询需求，分析哪些文档可能最相关
3. **记住所有可能相关的文档**：宁可多不可少，将所有可能相关的文档名称都记录下来
4. **确保回答准确性**：通过全面检索相关文档，确保回答的准确性和完整性

#### 检索流程说明

**完整的检索流程包括以下阶段：**

1. **召回阶段**：
   - 使用text-embedding-v4模型生成查询的embedding向量
   - 在Chroma向量数据库中检索指定文档的分块
   - 基于向量相似度召回最多20个chunk（可通过`--recall_top_k`参数调整）

2. **重排阶段**：
   - 使用qwen3-vl-rerank模型对召回结果重新排序
   - 返回3个最相关的chunk（可通过`--rerank_top_n`参数调整）

3. **滑动窗口召回**：
   - 为每个返回的chunk获取其相邻的上一个和下一个chunk
   - 构建完整的上下文信息，确保回答准确性

4. **去重处理**：
   - 如果返回的内容有重复信息，优先使用时间最新的chunk
   - 根据doc_update_time排序

#### 使用方法

**调用方式：**
```
execute_script(skill_id="rag_knowledge_base", script_name="rag_retrieve", args=["--documents", "2026金价预测.txt,黄金持仓数据.txt", "--query", "2026年金价预测"])
```

**参数说明：**
- `--documents`：文档名称列表，多个文档用逗号分隔（必需）
- `--query`：检索查询文本或关键词列表（必需）
- `--recall_top_k`：召回返回的chunk数量（可选，默认20）
- `--rerank_top_n`：重排返回的chunk数量（可选，默认3）


**返回结果字段说明：**
- `chunk_id`：分块唯一标识
- `category`：所属分类
- `doc_name`：文档名称
- `doc_update_time`：文档更新时间
- `chunk_content`：分块内容
- `chunk_index`：分块在文档中的索引
- `recall_score`：召回阶段的相似度分数（0-1）
- `rerank_score`：重排阶段的相关性分数（0-1）
- `context_chunks`：滑动窗口上下文
  - `previous`：上一个chunk的内容
  - `current`：当前chunk的内容
  - `next`：下一个chunk的内容


### 检索流程
完整的RAG检索流程包括以下阶段：

1. **检索前的准备**：
   - 调用`query_knowledge_base`脚本获取知识库信息
   - 分析查询需求，确定可能相关的文档
   - 收集所有可能相关的文档名称（宁可多不可少）

   - 返回3个最相关的chunk（可通过`--rerank_top_n`参数调整）

4. **滑动窗口召回**：

   - 根据doc_update_time排序
